# Implement: Presentation Decision

Use the `bundle-implementation` reasoning profile and the `ux-designer` skill.

## Objective

Analyze calendar form versus daily listing form for active sankalpa activity tracking, then implement the calmer, more useful presentation for the current app.

## Starting Hypothesis

A daily list grouped by week is likely calmer than a full calendar grid for this app because:

- active sankalpa cards already contain progress, filters, and actions
- phone screens need vertical clarity
- a calendar grid can look like a dashboard and hide labels behind color alone
- week-grouped rows make observed, missed, and pending states explicit

Validate this against the current UI and docs. If you choose a different presentation, document why.
